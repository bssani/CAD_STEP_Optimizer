# 사용법 (아주 쉬움)

STEP 파일 하나를 넣으면 → 웹용 3D 파일(GLB)로 바꾸고 → 품질을 자로 재서 → 가볍게 만들어 줍니다.

---

## 준비 (처음 한 번만)

`cadpipe-run` 폴더를 **통째로** 원하는 곳(예: 바탕화면)에 복사하세요.
폴더 안의 `cadpipe-run.exe` 가 실행 파일입니다. (폴더째로 옮겨야 작동해요. exe 하나만 빼면 안 됩니다.)

---

## 실행 방법 ① — 드래그 (제일 쉬움) 🖱️

1. 변환할 **`.step` (또는 `.stp`) 파일**을 마우스로 잡고
2. **`cadpipe-run.exe` 아이콘 위로 끌어다 놓으세요.**
3. 검은 창이 뜨고 잠깐(몇 초~몇 분) 돌아갑니다.
4. 끝나면 **`PASS`**(합격) 또는 `NEEDS REVIEW`(검토 필요)가 뜹니다.
   `Press Enter to close...` 가 보이면 **Enter** 를 누르면 창이 닫혀요.

## 실행 방법 ② — 옵션 주기 (터미널)

폴더 주소창에 `cmd` 입력 후 Enter → 검은 창에서:

```
cadpipe-run.exe  내모델.step  --web  --target 0.1
```

- `--web` : 웹용으로 **삼각형(폴리곤)까지 줄이기** (편차 목표 안에서 자동으로)
- `--target 0.1` : 허용 오차 0.1mm (이 안에 들면 합격)
- `--instance` : 똑같은 부품(나사 등)을 묶어 더 가볍게 (대신 개별 이름표는 사라짐)

---

## 결과는 어디에? 📁

방금 넣은 **STEP 파일 바로 옆에 `cadpipe_reports` 폴더**가 새로 생깁니다.
그 안 → `run_날짜_시간` 폴더 → 중요한 파일 2개:

| 파일 | 무엇 |
|---|---|
| **`run_summary.md`** | 한 장짜리 결과 요약 (합격 여부, 편차, draw call 등) — **이것부터 보세요** |
| **`final.glb`** | 웹에 올릴 **최종 3D 파일** |

(그 외 `production.glb`=다이어트 전 파일, `measure/`·`optimize/`=상세 측정 자료)

---

## 눈으로 확인하기 👀

`final.glb` 를 웹사이트 **https://sandbox.babylonjs.com** 화면에 끌어다 놓으면
3D로 빙글빙글 돌려볼 수 있어요. 부품 이름과 구조(트리)가 잘 있는지 확인하세요.

---

## 자주 묻는 것

- **설치할 게 있나요?** → **없습니다.** 변환·측정·다이어트에 필요한 Node와 gltf-transform까지
  폴더 안에 전부 들어있어요. 그냥 폴더째 복사하면 됩니다.
- **(드물게) "optimize phase skipped" 이라고 떠요** → 폴더가 일부만 복사되어 내장 Node가 빠졌을 때예요.
  폴더를 **통째로** 다시 복사하면 해결됩니다. (변환·측정 결과는 그래도 정상으로 나와요.)
- **창이 너무 빨리 닫혀요** → 드래그 대신 터미널(②)에서 실행하면 결과가 남습니다.
- **빨간 ERROR 가 떠요** → STEP 파일이 깨졌거나 경로에 문제일 수 있어요. 다른 STEP으로 시도해 보세요.

# CAD STEP → GLB 변환·검증 도구 — 사용법

CATIA 등에서 뽑은 **STEP 파일**을 웹 뷰어(Babylon.js)용 **GLB**로 바꾸고,
변환이 잘 됐는지 **숫자로 검증**해 주는 도구입니다.
설치할 게 없습니다 — 파이썬·Node·gltf-transform이 폴더 안에 전부 들어있어요.

---

## 0. 준비 (처음 한 번만)

`cadpipe-run` **폴더를 통째로** 원하는 곳(바탕화면 등)에 복사하세요.
👉 폴더 안 **`cadpipe-run.exe`** 가 실행 파일입니다.

> ⚠️ **폴더째로** 옮기세요. exe 하나만 빼내면 작동하지 않습니다.

폴더 구성:
```
cadpipe-run\
  cadpipe-run.exe          ← 실행 파일
  테스트하기.bat            ← 동작 확인용 (더블클릭)
  sample_DemoBracket.step  ← 테스트용 샘플
  사용법.md / README.md     ← 설명서
  _internal\               ← 엔진(node·gltf-transform 등). 건드리지 마세요
```

---

## 1. 먼저 "이 PC에서 도는지" 확인 — `테스트하기.bat`

새 PC(예: 회사 컴퓨터)에서는 **항상 이것부터** 하세요.

1. 폴더 안 **`테스트하기.bat` 더블클릭**
2. 잠시 후(몇 초~몇 분) 검은 창에 결과가 나옴
3. **`[run ...] PASS`** 가 보이면 → **이 PC에서 정상 작동!** ✅

안 되면 → 아래 **6. 문제 해결** 참고.

---

## 2. 실제 사용 — STEP 파일을 드래그 🖱️

변환할 **`.step`(또는 `.stp`) 파일**을 마우스로 잡아 → **`cadpipe-run.exe` 아이콘 위에 떨어뜨리세요.**

검은 창이 뜨고 끝나면 이렇게 나옵니다:
```
Input : C:\...\내모델.step
Output: C:\...\cadpipe_reports      ← 결과가 여기 생겨요
[run ...] PASS                       ← 합격!
  final P95: 0.0062 mm (target 0.1) -> within
```
`Press Enter to close...` 가 보이면 **Enter**.

---

## 3. 옵션 주기 (터미널)

폴더 주소창에 `cmd` 입력 후 Enter → 검은 창에서:
```
cadpipe-run.exe  내모델.step  --web  --target 0.1
```

| 옵션 | 뜻 | 언제 |
|---|---|---|
| `--web` | 웹용으로 **삼각형(폴리곤)까지 줄이기** (오차 범위 안에서 자동) | 웹에 올릴 때 |
| `--target 0.1` | 허용 오차 0.1mm (이 안에 들면 합격) | 정밀도 기준 정할 때 |
| `--instance` | 똑같은 부품(나사 등) 묶어 더 가볍게 | draw call 더 줄일 때 (※개별 이름표는 사라짐) |
| `--draw-call-budget 200` | draw call 예산 초과 시 경고 표시 | 성능 기준 정할 때 |
| (옵션 없음) | 안전 기본값: 구조 100% 보존 + 압축 | 평소 |

---

## 4. 결과 보는 법 📁

넣은 STEP 파일 **바로 옆에 `cadpipe_reports` 폴더**가 생깁니다.
그 안 → `run_날짜시간` 폴더 → 중요한 파일:

| 파일 | 무엇 | 어떻게 |
|---|---|---|
| **`run_summary.md`** | 한 장짜리 결과 요약 | 메모장/뷰어로 열기 — **이것부터** |
| **`final.glb`** | 웹에 올릴 **최종 3D 파일** | 아래 5번으로 확인 |
| `production.glb` | 다이어트 전 변환 결과 | 참고용 |
| `manifest.json` | 전체 수치 + 도구 버전 (재현용) | 문의 시 첨부 |
| `measure/` `optimize/` | 상세 측정·최적화 자료 | 깊이 볼 때 |

### `run_summary.md` 에서 보는 핵심 숫자
| 항목 | 뜻 | 정상 |
|---|---|---|
| **P95 deviation** | 원본과의 오차(95% 지점) | 목표(예: 0.1mm) 이내 |
| **draw calls** | 웹 성능 핵심 지표 (그리는 횟수) | 적을수록 좋음 |
| **rendered triangles** | 삼각형(폴리곤) 수 | 적을수록 가벼움 |
| **names preserved** | 부품 이름 보존 | `4/4` 처럼 전부 |
| **hierarchy preserved** | 조립 구조(트리) 보존 | `True` |
| **topology** | 면 누락·뒤집힘 등 | `0` |
| **Overall** | 종합 합격 여부 | `✅ PASS` |

## 5. 눈으로 확인 — Babylon Sandbox 👀

`final.glb` 를 웹사이트 **https://sandbox.babylonjs.com** 화면에 **끌어다 놓으면**
3D로 빙글빙글 돌려볼 수 있어요. **부품 이름과 구조(트리)가 잘 보이는지** 확인하세요.

## 6. 문제 해결 (Troubleshooting)

| 증상 | 원인 / 해결 |
|---|---|
| **백신/“Windows의 PC 보호” 경고로 안 열림** | 처음 보는 exe라 보안프로그램이 막는 것. → "추가 정보 → 실행", 또는 exe 우클릭 → 속성 → **차단 해제(Unblock)**. 그래도 막히면 **회사 IT에 화이트리스트 요청** |
| **창이 번쩍하고 바로 닫힘** | 드래그 대신 **`테스트하기.bat`**(자동으로 멈춤) 또는 터미널(3번)에서 실행 |
| **"optimize phase skipped" 표시** | 폴더가 일부만 복사돼 내장 Node가 빠진 것. → 폴더를 **통째로** 다시 복사 (변환·측정 결과는 그래도 정상) |
| **빨간 `ERROR` 표시** | STEP 파일이 깨졌거나 경로 문제일 수 있음. 다른 STEP으로 시도, 또는 아래 7번대로 문의 |
| **결과 저장이 안 됨** | STEP이 **읽기전용 네트워크 드라이브**에 있을 때. → STEP을 **로컬(바탕화면 등)에 복사한 뒤** 실행 |
| **부품 이름이 깨져 보임(한글 등)** | 인코딩 이슈일 수 있음. `run_summary.md`와 함께 문의 |

---

## 7. 문제가 있어 문의할 때 — 이것만 보내주세요

**🟢 잘 됐을 때** (수치를 검토받고 싶을 때)
→ 결과 폴더의 **`run_summary.md` + `manifest.json`** 파일 2개

**🔴 안 됐을 때**
→ **막힌 화면 캡쳐 1장** (`Win+Shift+S` 로 캡쳐) + "어디서 막혔는지" 한 줄
   - 창이 안 뜸 / 백신 경고 / 빨간 ERROR / optimize skipped 중 무엇인지

> 🔒 실파일이 회사 기밀이면 `manifest.json` 의 **파일명·SHA 해시 줄만 지우고** 주셔도 분석엔 지장 없습니다. (`final.glb` 같은 형상 파일은 안 보내셔도 돼요.)

## 부록 C. 알아둘 한계 (정직하게)
- 더티 CAD의 **면 뒤집힘·누락**은 **표시(플래그)만** 하고 자동 수리는 안 함.
- 기준(정답) 메시는 초고밀도 근사값 → 편차는 "기준 대비" 값.
- **부품 간 병합은 안 함**(구조 보존 우선). draw call이 예산을 크게 넘으면 **별도 검토 대상**으로 표시만.
- `--instance` 사용 시 반복 부품의 **개별 이름표는 사라짐**(부품 종류 이름은 유지).
